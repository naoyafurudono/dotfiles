# C4 コードレベル設計の指針

C4モデルの Level 4（コードレベル）で設計を行う際、以下の3原則に従う。

## 1. フレームワークの慣習に従う

使用しているフレームワークやミドルウェア（Ruby on Rails, MySQL 等）が持つ規約・慣習を優先する。フレームワークが提供するレールに乗ることで、設計の一貫性と保守性を確保する。

## 2. 既存コードのパターンを尊重する

同様の目的を持つ実装が既存コードベースにある場合、そのパターンに合わせる。独自のやり方を持ち込む前に、既存の解法を確認し踏襲する。

## 3. 変更をできる限り小さくする

一度の変更のスコープを最小限に保つ。大きな変更は分割し、レビュー・理解・ロールバックが容易な単位にする。
