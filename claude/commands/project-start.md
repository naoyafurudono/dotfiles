プロジェクトの作業開始セットアップを行います。以下を順に実施してください。

## 1. プロジェクト状態の確認

- CLAUDE.md を読み、プロジェクトの概要・進行中タスクを把握する
- `docs/designs/` の設計書・タスクファイルがあれば読み込む
- `repos/` にクローン済みのリポジトリを確認する

## 2. タスクの最新状態を同期

タスクファイル（`docs/designs/*-tasks.md`）がある場合：

- `repos/` 配下の各リポジトリで `git status` / `git branch` を確認
- GHES の PR 状態を確認: `GH_HOST=git.pepabo.com gh pr list -R colorme/<repo>`
- タスクファイルの状態が実態と乖離していれば更新する

## 3. 状況サマリーを出力

以下の形式で現在の状況を報告してください：

```
## プロジェクト: <名前>

### 進捗サマリー
- 完了: N タスク
- 進行中: N タスク（ブランチ/PR 情報付き）
- 未着手: N タスク

### 直近の作業
- <最後に作業した内容>

### 次にやること
- <推奨する次のアクション>
```

## 4. 足りないリポジトリの確認

設計書やタスクに記載されている実装先リポジトリが `repos/` にクローンされていない場合は、クローンするか確認してください。
